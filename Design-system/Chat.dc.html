What's the cleanest way to retry a flaky fetch with backoff?
JM
An
Use capped exponential backoff with full jitter so retries don't thundering-herd. The key ideas:
  • Double the delay each attempt, capped at ~15s
  • Multiply by Math.random() for jitter
  • Stop after a max attempt count and surface the error
Can you give me the TypeScript?
JM
An
Sure — here it is, fully typed:
typescript
export async function retry<T>(fn: () => Promise<T>, max = 5) {
  for (let i = 0; ; i++) {
    try { return await fn(); }
    catch (e) {
      if (i >= max) throw e;
      const d = Math.min(2 ** i * 1000, 15000);
      await sleep(Math.random() * d);
    }
  }
}
