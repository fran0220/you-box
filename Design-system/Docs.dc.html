Quickstart

YouBox exposes a single OpenAI-compatible endpoint that routes to 300+ models. If you already use the OpenAI SDK, you're three lines away — change the base URL, drop in your key, and pass a provider-prefixed model slug.

Base URL

All requests go to one host. Point any OpenAI-compatible client at it:

http
https://api.youbox.dev/v1

Authentication

Authenticate with a bearer token. Create a key in API keys and pass it in the Authorization header. Keep it server-side — never ship it to a browser.

Your first call

Send a chat completion. Switch providers by changing only the model string:

curl https://api.youbox.dev/v1/chat/completions \
  -H "Authorization: Bearer $YOUBOX_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "anthropic/claude-opus-4.6",
    "messages": [{"role": "user", "content": "Hello!"}],
    "stream": false
  }'

Request parameters

The body follows the OpenAI Chat Completions schema, with a few YouBox extensions for routing.

FieldTypeDescription
model *stringProvider-prefixed slug, e.g. openai/gpt-5.2.
messages *arrayConversation history as role/content objects.
streambooleanStream tokens as server-sent events. Default false.
temperaturenumberSampling temperature, 0–2. Default 1.
routestringYouBox routing strategy: cost, latency, or quality.
fallbacksarrayOrdered model slugs to try if the primary fails.

Response

json
{
  "id": "chatcmpl-7f3a9c",
  "model": "anthropic/claude-opus-4.6",
  "choices": [{ "message": { "role": "assistant", "content": "Hi!" } }],
  "usage": { "prompt_tokens": 9, "completion_tokens": 2 },
  "yb_provider": "anthropic-direct"
}

Error codes

CodeMeaning
401Missing or invalid API key.
429Rate limit or budget exceeded — back off and retry.
502All upstream providers failed after failover.
